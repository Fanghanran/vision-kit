# 摄像头管理模块 v2 设计

> 版本: v2 | 日期: 2026-07-09

## 一、现状

Cameras.vue 是卡片网格布局，有添加/编辑弹窗 + 启停/删除操作 + 5 秒轮询。存在以下不足：

| 缺失 | 说明 |
|---|---|
| 统计概览 | 无总数/在线/离线统计 |
| 搜索筛选 | 摄像头多了只能翻卡片找 |
| 详情查看 | 点编辑才看到配置，无单独详情面板 |
| 运行指标 | 卡片只显示 FPS/告警/队列，缺运行时长、重连次数、延迟 |
| 编辑预填 | openEditDialog 不加载当前配置，始终重置为默认值 |
| 摄像头名称 | 卡片只显示 camera_id，没有显示 camera_name |
| 批量操作 | 无法批量启停 |

## 三、实施计划

### 3.1 后端新增

| 端点 | 权限 | 说明 |
|---|---|---|
| GET /api/cameras/stats | viewer+ | 摄像头统计：总数/在线/离线/告警中 |
| GET /api/cameras/{id} | viewer+ | 单个摄像头详情（含配置脱敏） |

### 3.2 前端改造

| 文件 | 操作 | 说明 |
|---|---|---|
| views/Cameras.vue | 重写 | 统计卡片 + 搜索筛选 + 右侧抽屉 + 优化卡片 |
| api/cameras.ts | 修改 | 新增 getCameraStats、getCameraDetail |
| stores/cameras.ts | 修改 | 新增 stats、detail 状态 |

### 3.3 卡片信息增强

```
现有                          新增
─────────────────────────────────────
camera_id                      camera_name（优先显示名称）
status + 图标                  source_type 标签
FPS                            GPU延迟 ms
total_alerts                   今日告警
queue_size                     uptime_seconds 格式化
-                              重连次数（如有）
```
