# 视频监控面板 — 设计书

## 1. 定位

独立于 Dashboard 的专用视频监控页面，用于安保人员实时查看多路摄像头画面，支持灵活布局、画面叠加检测信息、历史回放。

核心体验：
- **所见即所得**：画面实时、检测框实时叠加
- **灵活布局**：1/4/9/16 分屏，可拖拽调整
- **快速切换**：点击放大、双击全屏、右键菜单
- **回放溯源**：时间轴拖动查看历史录像

## 3. 功能设计

### 3.1 布局系统

| 布局 | 格子数 | 适用场景 |
|------|--------|---------|
| 1×1 | 1 | 专注查看单路（放大/全屏） |
| 2×2 | 4 | 日常监控，4 路同屏 |
| 3×3 | 9 | 中型监控站 |
| 4×4 | 16 | 大型监控墙 |

**交互**：
- 顶栏按钮切换布局
- 切换时保留已有分配（如从 2×2 切到 3×3，已分配的 4 路保持不变，新增 5 个空位）
- 空位显示"点击选择摄像头"占位符

### 3.2 摄像头分配

**方式**：
- 点击空位 → 弹出摄像头选择下拉框
- 拖拽左侧摄像头列表中的项 → 拖到目标格子
- 右键格子 → 上下文菜单（替换/移除/全屏/截图）

**摄像头列表**（左侧或顶部下拉）：
```
┌──────────────┐
│ 🟢 cam_01    │
│ 🟢 cam_02    │
│ 🔴 cam_03    │  ← 离线不可拖入
│ 🟡 cam_04    │  ← 降级可拖入
└──────────────┘
```

### 3.3 视频流方案

| 方案 | 协议 | 延迟 | 浏览器支持 | 适用场景 |
|------|------|------|-----------|---------|
| MJPEG | HTTP multipart | 0.5-2s | 原生支持 | 简单部署，兼容性好 |
| HLS | HTTP | 2-5s | 原生/插件 | 回放，CDN 友好 |
| WebSocket JPEG | WS binary | 0.1-0.5s | 需 JS 解码 | 低延迟实时 |

**选择**：WebSocket JPEG（实时）+ HTTP MP4（回放）

**实时流**：
- 后端每帧 JPEG 编码，通过 WebSocket 推送
- 前端 `<img>` 或 Canvas 渲染
- 延迟 < 500ms

**回放流**：
- 后端提供 `/api/cameras/{id}/replay?start=TIMESTAMP&end=TIMESTAMP`
- 返回 MP4 视频文件
- 前端 `<video>` 标签播放

### 3.4 画面叠加

实时画面上叠加检测信息：

```
┌─────────────────────────┐
│  ┌─────┐                │
│  │person│ ← 检测框+标签   │
│  │ 0.89 │ ← 置信度       │
│  └─────┘                │
│      ·                  │
│      · ← 轨迹线          │
│      ·                  │
│  ┌──────────┐           │
│  │  car 0.76│           │
│  └──────────┘           │
│ cam_01 | 14:32:15 | 5fps │ ← 信息栏
└─────────────────────────┘
```

**叠加内容**：
- 检测框（Bounding Box）+ 类别标签 + 置信度
- 追踪轨迹（最近 N 个位置的连线）
- 摄像头信息栏（ID、时间、FPS）
- 告警高亮（触发告警时边框闪烁红色）

**控制**：
- 叠加层开关（显示/隐藏检测框）
- 轨迹长度控制（0-100 帧）

### 3.5 回放功能

**时间轴**：
```
┌────────────────────────────────────────────┐
│ ◀◀  ◀  ▶  ▶▶  │ 速度: 1x 2x 4x 8x       │
│ ━━━━━━━━━━━━━●━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│ 12:00        14:32        18:00   24:00   │
└────────────────────────────────────────────┘
```

**交互**：
- 拖动时间轴滑块 → 跳转到指定时间
- 播放/暂停/快进/快退
- 速度控制（1x/2x/4x/8x）
- 时间范围选择（最近1小时/今天/自定义）

**后端 API**：
```
GET /api/cameras/{id}/replay?start=TIMESTAMP&end=TIMESTAMP
→ 返回 MP4 视频流

GET /api/cameras/{id}/timeline?date=YYYY-MM-DD
→ 返回该天有录像的时间段列表（用于时间轴高亮）
```

### 3.6 快捷操作

| 操作 | 方式 | 说明 |
|------|------|------|
| 放大单路 | 点击格子 | 该路占满整个面板 |
| 全屏 | 双击格子 | 浏览器全屏显示该路 |
| 截图 | 右键 → 截图 | 保存当前帧到本地 |
| 替换摄像头 | 右键 → 替换 | 弹出选择框 |
| 移除 | 右键 → 移除 | 清空该格子 |
| 返回网格 | ESC 或点击返回 | 从放大/全屏退回网格 |

## 4. 后端新增 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/cameras/{id}/stream` | GET | MJPEG 实时流（备选方案） |
| `/api/cameras/{id}/replay` | GET | 历史录像 MP4 |
| `/api/cameras/{id}/timeline` | GET | 某天有录像的时间段 |
| `/ws/video/{id}` | WS | WebSocket JPEG 实时帧推送 |

### 4.1 WebSocket 视频流协议

```
客户端 → ws://host:8080/ws/video/cam_01

服务端推送：
  binary: JPEG 编码的帧数据（每帧一条消息）

  前 8 字节为帧头（可选）：
    [0-3]: 帧序号 (uint32, big-endian)
    [4-7]: 时间戳 (uint32, 秒级 Unix 时间)
    [8-N]: JPEG 数据
```

### 4.2 回放 API

```
GET /api/cameras/{id}/replay?start=1625000000&end=1625003600

响应：
  Content-Type: video/mp4
  Content-Disposition: inline; filename="cam_01_1625000000.mp4"
  
  二进制 MP4 数据
```

## 5. 前端组件结构

```
views/
  Monitor.vue              # 监控面板主页面
components/
  monitor/
    MonitorGrid.vue        # 可调整布局的网格容器
    VideoCell.vue          # 单个视频格子（实时/回放/空位）
    VideoOverlay.vue       # Canvas 检测框叠加层
    TimelineBar.vue        # 回放时间轴
    CameraSelector.vue     # 摄像头选择下拉
    LayoutPicker.vue       # 布局选择器（1/4/9/16）
```

### 5.1 MonitorGrid

```
props:
  - layout: { cols: number, rows: number }
  - cells: Array<{ cameraId: string, mode: 'live' | 'replay' }>

功能：
  - CSS Grid 布局
  - 拖拽排序（HTML5 Drag & Drop）
  - 点击/双击事件
  - 右键菜单
```

### 5.2 VideoCell

```
props:
  - cameraId: string
  - mode: 'live' | 'replay' | 'empty'
  - overlay: boolean
  - replayTime?: number

功能：
  - 实时模式：WebSocket 接收 JPEG，渲染到 <img> 或 Canvas
  - 回放模式：<video> 标签播放 MP4
  - 空位模式：显示"点击选择"占位符
  - 信息栏叠加（ID、时间、FPS）
  - 告警边框闪烁
```

### 5.3 VideoOverlay

```
props:
  - detections: Array<{ bbox, class_name, confidence }>
  - tracks: Array<{ trajectory }>
  - frameWidth: number
  - frameHeight: number

功能：
  - Canvas 绘制检测框
  - Canvas 绘制轨迹线
  - 坐标映射（模型坐标 → 显示坐标）
```

## 6. 性能考虑

| 场景 | 策略 |
|------|------|
| 16 路同时显示 | WebSocket JPEG，每路 5fps，总 80 帧/秒，带宽约 10MB/s |
| 检测框叠加 | Canvas 独立层，不重绘视频帧 |
| 回放拖动 | 预加载关键帧，拖动时跳转到最近关键帧 |
| 内存管理 | 离开页面时断开所有 WebSocket，释放 Canvas |
| 网络降级 | 自动降低帧率或分辨率（16 路时降到 2fps） |
