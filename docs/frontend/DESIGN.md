# SentinelMind 前端设计书

> 技术栈：Vue 3 + TypeScript + Vite + Element Plus + Pinia

---

## 1. 定位与目标

SentinelMind 前端是多路视频智能分析系统的监控面板，核心用户是**值班安保人员**和**系统运维人员**。

**核心体验目标**：
- 告警来了**立刻看到**（WebSocket 实时推送，不轮询）
- 处理告警**一键操作**（确认/标记误报，3 秒内完成）
- 系统状态**一目了然**（摄像头在线、GPU 负载、队列积压）

**不做**：
- 不做视频实时预览（第一版，后续可扩展）
- 不做规则在线编辑（YAML 文件编辑更灵活）
- 不做用户权限管理（单用户 + Token 认证）

## 4. WebSocket 实时推送

### 4.1 连接管理

```typescript
// 连接地址：ws://host:port/ws?token=<api_token>
// 自动重连：断开后 3s → 6s → 12s → 30s 指数退避
// 心跳：服务端 30s ping，客户端 pong
```

### 4.2 消息类型

| type | 触发时机 | 前端处理 |
|------|---------|---------|
| `new_alert` | 新告警生成 | 仪表盘插入实时流 + 提示音 + 列表顶部新行 |
| `alert_status` | 告警状态变更 | 更新列表/详情中的状态标签 |
| `system_status` | 每 10s 定时 | 更新仪表盘统计卡片 |
| `camera_status` | 摄像头上线/下线 | 更新摄像头状态指示灯 |

### 4.3 提示音

新告警到达时播放短促提示音（区分紧急/非紧急）：
- 紧急/高风险：连续短促蜂鸣
- 中/低风险：单次轻响

## 5. 认证与 Token

### 5.1 Token 输入

- 首次访问时弹出 Token 输入对话框
- Token 存储在 `localStorage`，有效期跟随后端配置
- 顶栏显示 Token 状态（已认证 ✅ / 未认证 ❌）
- 支持"退出"（清除 Token）

### 5.2 认证失败处理

- API 返回 401/403 → 自动弹出 Token 输入对话框
- WebSocket 连接失败（token 无效）→ 断开并提示重新输入

// styles/variables.scss

  --va-bg-card: #FFFFFF;
  --va-text-primary: #303133;
  --va-text-secondary: #909399;

**状态标签**：使用 Element Plus `el-tag` 组件，不同状态不同颜色和图标

**行点击**：整行可点击跳转详情，hover 时背景色加深 `cursor: pointer`

## 9. 响应式设计

|------|------|

| 768px ~ 1199px | 侧边栏折叠为图标 + 卡片单列 |

## 10. 性能考虑

| 场景 | 策略 |
|------|------|
| 告警列表 | 虚拟滚动（Element Plus el-table-v2），支持万级数据 |
| 截图加载 | 懒加载 + 缩略图（后端可提供 ?size=thumb 参数） |
| WebSocket 消息 | 防抖处理（300ms），避免频繁 DOM 更新 |
| 图表数据 | 最多保留 1000 个数据点，旧数据滑窗丢弃 |
| 首屏加载 | Vite code splitting + 路由懒加载 |

## 11. 部署方式

### 9.1 开发模式

```bash
cd frontend
npm install
npm run dev          # Vite dev server on :5173
# 代理 API 请求到后端 :8080
```

### 9.2 生产构建

```bash
npm run build        # 输出到 dist/
```

构建产物（`dist/`）复制到后端 `static/` 目录，由 FastAPI 的 StaticFiles 中间件 serve：

```
GET /           → index.html
GET /assets/*   → 静态资源
GET /api/*      → REST API
GET /ws         → WebSocket
```

### 9.3 Vite 代理配置

```typescript
// vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/api': 'http://localhost:8080',
      '/ws': { target: 'ws://localhost:8080', ws: true },
      '/health': 'http://localhost:8080',
    },
  },
})
```
