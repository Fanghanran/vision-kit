T 线程模型

  • CameraThread × N — 每路独立线程，断线不影响其他路
  • InferenceThread × 1 — 专用GPU线程，batch合并提升吞吐
  • ActionThread × 1 — 顺序处理，LLM异步调用不阻塞告警

F 容错策略

  • 摄像头断线 — 指数退避重连，5分钟未恢复告警
  • GPU崩溃 — 跳帧+追踪兜底，连续100帧降级为仅录制
  • LLM/Redis失败 — 降级为裸规则告警/内存缓存，不中断

S 技术栈

FastAPI Ultralytics YOLO BoT-SORT FFmpeg OpenAI API ChromaDB SQLite Vue 3 systemd
设计原则：每个组件独立生命周期 | 有界队列丢旧帧保实时性 | 单点故障不传播 | 能降级就降级
